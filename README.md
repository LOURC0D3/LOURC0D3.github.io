# blog.lourcode.kr

[Astro](https://astro.build) + [Erudite](https://github.com/jktrn/astro-erudite) blog with Notion as the editing surface. Vulnerability writeups, CTF notes, security research.

## Stack

- **Generator**: Astro 6 + MDX
- **Theme**: [astro-erudite](https://github.com/jktrn/astro-erudite)
- **Editing**: Notion database (button-triggered publish via `repository_dispatch`)
- **Hosting**: GitHub Pages
- **Domain**: blog.lourcode.kr (CNAME)

## Local development

```sh
bun install
bun run dev      # http://localhost:1234

```

### Triggering deploy

The HTML button embedded in the Notion blog page sends a `repository_dispatch` event (`RUN_WORKFLOW_DISPATCH`) which runs `.github/workflows/pages-deploy.yml`. The workflow re-runs the importer, commits changes, builds the Astro site, and deploys to GitHub Pages.

## MDX magic code blocks

Vulnerability writeups often contain payloads (`{{7*7}}`, `<script>...</script>`) that look like template/JSX syntax. The importer outputs **plain `.md`** by default so these stay intact.

To use Erudite's [Callout component](src/components/callout.astro), open a code block in Notion with a special language. The importer detects it and unwraps to MDX:

````markdown
```callout-warning CVE-2024-1234
Vulnerable versions: 1.0 ~ 2.3.5
```
````

becomes:

```mdx
import Callout from '@/components/callout.astro'

<Callout variant="warning" title="CVE-2024-1234">
Vulnerable versions: 1.0 ~ 2.3.5
</Callout>
```

Supported languages:

- `mdx`, `jsx`, `astro` â€” fence is stripped, content is inserted as raw MDX
- `callout-{variant}` â€” wraps content in `<Callout variant="{variant}">`. Variants: `note`, `tip`, `warning`, `danger`, `important`, `definition`, `theorem`, `lemma`, `proof`, `corollary`, `proposition`, `axiom`, `conjecture`, `notation`, `remark`, `intuition`, `recall`, `explanation`, `example`, `exercise`, `problem`, `answer`, `solution`, `summary`
- Optional title after a space: `` ```callout-warning Some title ``

When **any** magic block is used, the importer writes the post as `.mdx` and auto-escapes `{` â†’ `\{`, `<` â†’ `\<` in prose (outside code blocks) to keep payloads safe.

Regular code blocks (` ```python `, ` ```c `, etc.) are left untouched; payloads inside them never need escaping.
